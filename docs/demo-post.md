# 【社会服务赛道】安心小屏 Demo - 旧手机改造老人陪伴助手（附在线体验）

## 1. Demo 简介

安心小屏是一个纯前端 PWA 网页应用，主要用于把闲置旧安卓手机改造成老人桌面陪伴屏。它不需要安装 App，浏览器打开就能使用，适合放在老人家中的桌面、床头或客厅，作为一个大字版、少操作的陪伴入口。

这个 Demo 面向独居老人、留守老人，以及在外地工作、希望远程照护父母的子女。设计时优先考虑“老人敢不敢点、看不看得清、出问题能不能马上联系家人”，所以界面尽量使用大字体、大按钮、清晰状态提示和语音入口。

目前 Demo 主要包含以下功能：

1. 提醒系统：子女可以在子女端远程设置提醒，老人端到点后弹出大字提醒，老人点击确认后，子女端可以看到状态同步。
2. AI 问助手：支持本地语音识别入口和文字输入，老人可以询问天气、时间、常识等简单问题，页面用更适合老人阅读的方式展示回答。
3. 视频通话：演示从子女发起呼叫、老人端收到来电、接听后进入通话中的完整流程，并在界面上实时显示摄像头和麦克风的隐私状态。

## 2. Demo 创作思路

这个想法来自我对农村留守老人使用智能手机的观察。很多老人不是完全不需要手机，而是智能手机里的入口太多、字太小、提示太复杂。一旦点错页面，他们很难自己退回来，也会担心“把手机弄坏”。子女在外打工或异地生活时，想提醒父母吃药、喝水、休息，往往只能靠电话反复沟通。

我想先解决三个很具体的问题：

- 老人容易忘记吃药、复诊、喝水等日常事项，子女又很难每次准时电话提醒。
- 智能手机操作复杂，老人常常不敢用，很多功能对他们来说藏得太深。
- 独居老人缺少陪伴，也缺少一个醒目的紧急求助入口。

技术方案上，我选择把旧手机改造成一个“固定摆放的小屏”，而不是重新设计硬件。这样可以利用家里闲置的安卓手机，降低硬件成本。PWA 的方式也比较适合 Demo 阶段：打开浏览器就能体验，不需要安装新 App，也方便部署在线演示。

在交互上，安心小屏尽量减少层级。老人端首页只保留几个大按钮，常用功能直接可见；提醒、来电、隐私状态都用大字和明确提示展示；语音输入作为补充入口，让不熟悉打字的老人也能尝试使用。

## 4. TRAE 实践过程

这次 Demo 的开发过程主要分为五步：创意分析、技术选型、原型开发、功能完善、部署上线。

创意分析阶段，我先围绕“旧手机能不能变成适老化陪伴屏”拆解使用场景，把老人端和子女端分开考虑。老人端强调少操作、看得清、能求助；子女端强调能添加提醒、查看状态、发起联系。

技术选型阶段，我确定使用 HTML、CSS 和原生 JavaScript 做纯前端 PWA Demo。这样项目结构更轻，适合快速迭代，也方便直接部署到 Vercel。视频通话和信令服务器作为可选能力保留在项目中，不影响静态页面体验。

原型开发阶段，我先完成老人端首页、子女端概览、提醒列表和基础状态管理。随后补充 AI 问助手、娱乐陪伴、紧急联系和视频通话流程，让 Demo 能覆盖从日常陪伴到远程照护的完整故事。

功能完善阶段，我重点打磨了大字体、大按钮、提醒确认、隐私状态提示和截图展示。相比追求复杂功能，我更希望这个 Demo 能清楚表达社会公益方向的价值：用已有设备和简单网页，给老人一个更容易使用的数字入口。

部署上线阶段，本次对话完成了 Git 仓库初始化、README、Vercel 静态部署配置和初赛 Demo 帖内容准备。后续我会把项目部署到 Vercel，并把在线体验地址补充到帖子中。
